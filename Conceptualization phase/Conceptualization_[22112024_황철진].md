# 1. Business purpose
  
Smart Medication & Supplement Management System의 목적은 사용자가 일상적으로 복용하는 의약품 및 영양제를 보다 안전하고 효율적으로 관리할 수 있도록 지원하는 것이다.
현대 사회에서 많은 사람들은 처방약, 일반의약품, 그리고 다양한 영양제를 함께 복용하고 있다. 그러나 복용 시간과 방법을 정확히 기억하지 못하거나, 제품에 대한 신뢰할 수 있는 정보를 얻기 어려운 경우가 많다. 이러한 문제는 복용 누락, 잘못된 복용 방법, 그리고 부작용이나 약물 간 상호작용에 대한 인지 부족으로 이어질 수 있다.
본 시스템은 이러한 문제를 해결하기 위해 복약 일정 관리, 복용 기록 저장, 그리고 의약품 및 영양제에 대한 종합적인 정보 제공 기능을 통합한 플랫폼을 제공한다. 사용자는 정해진 시간에 알림을 받고 복용 여부를 기록할 수 있으며, 각 제품의 효능, 복용 방법, 주의사항 등의 정보를 쉽게 확인할 수 있다.
또한, 사용자 리뷰 기능을 통해 실제 복용 경험과 효과, 부작용 등에 대한 정보를 공유할 수 있도록 하여, 공식 정보뿐만 아니라 사용자 기반의 실질적인 정보를 함께 제공한다.
궁극적으로 본 시스템은 사용자의 복약 순응도를 향상시키고, 보다 안전한 복용 환경을 조성하며, 일상적인 건강 관리를 보다 편리하고 체계적으로 지원하는 것을 목표로 한다.

# 2. System context diagram
```mermaid
    flowchart TB
    System(시스템)
    Users[사용자]
    Server[서버]
    Admin[관리자]

    Users ---->|회원가입 / 로그인 요청\n약 정보 조회\n복용 기록 입력\n리뷰 작성| System
    System ---->|로그인 승인\n복약 알림\n약 정보 제공\n리뷰 결과 제공| Users

    System ---->|사용자 정보 / 약 정보 / 리뷰 데이터 요청| Server
    Server ---->|데이터 저장 및 조회 결과 반환| System

    Admin ---->|사용자 관리 / 리뷰 관리 요청| System
    System ---->|관리 결과 / 신고 정보 제공| Admin
```
System : Smart Medication & Supplement Management System으로, 복약 관리, 정보 제공, 사용자 관리 등의 핵심 기능을 수행하는 중심 시스템이다.

Users : 시스템을 사용하는 일반 사용자로, 회원가입, 로그인, 약 정보 조회, 복용 기록 입력, 리뷰 작성 등의 기능을 수행한다.

Server (DB Server) : 사용자 정보, 약 정보, 리뷰 데이터를 저장하고 관리하는 외부 시스템으로, 시스템의 요청에 따라 데이터를 제공하거나 저장한다.

Administrator : 시스템을 관리하는 주체로, 사용자 관리 및 리뷰 관리 등의 관리 기능을 수행한다.

# 5. Problem statement
- Overview
본 시스템은 사용자가 약 및 영양제를 보다 편리하게 관리하고, 복용 정보를 확인할 수 있도록 지원하는 것을 목적으로 한다. 또한 약/영양제에 대한 정보 제공과 사용자 리뷰 기능을 통해 보다 안전하고 효율적인 복약 환경을 제공하고자 한다.
이 시스템은 다음과 같은 목적을 달성해야 한다.

----
- 정확한 복약 일정 관리
- 사용자와 관리자의 역할 구분
- 사용자 및 복약 정보의 데이터베이스화
- 약/영양제 정보 및 리뷰 관리 기능 제공
----

- Problem #1

사용자가 약 복용 시간을 놓치거나 복용 여부를 정확히 기억하지 못하는 문제가 발생할 수 있다.
이로 인해 복약 효과가 감소하거나 건강에 부정적인 영향을 줄 수 있다.
따라서 복약 일정 관리와 알림 기능이 정확하게 동작해야 한다.

- Problem #2

약 및 영양제에 대한 정보가 여러 곳에 분산되어 있어 사용자가 신뢰할 수 있는 정보를 얻기 어렵다.
잘못된 정보를 기반으로 복용할 경우 부작용 등의 문제가 발생할 수 있다.
따라서 정확하고 신뢰성 있는 정보를 제공하는 시스템이 필요하다.

- Problem #3

사용자가 약 정보를 직접 등록하고 복용 일정을 설정하는 과정이 번거로울 수 있다.
특히 고령자의 경우 시스템 사용이 어려워 사용을 포기할 가능성이 있다.
따라서 간단하고 직관적인 사용자 인터페이스 설계가 필요하다.

- Problem #4

사용자 리뷰 기능을 제공할 경우 부정확하거나 악의적인 정보가 포함될 수 있다.
이는 다른 사용자에게 잘못된 정보를 전달할 수 있다.
따라서 관리자에 의한 리뷰 관리 기능이 필요하다.

- Problem #5

사용자 정보 및 복약 기록 등 민감한 데이터가 시스템에 저장되기 때문에 개인정보 보호 문제가 발생할 수 있다.
또한 시스템이 해킹될 경우 개인정보 유출로 이어질 위험이 있다.
따라서 보안 및 데이터 보호가 중요하게 고려되어야 한다.

- Problem #6

본 시스템은 모바일 환경에서 사용되는 것을 고려하고 있으나, 다양한 운영체제 환경이 존재한다.
특정 플랫폼에서만 동작할 경우 일부 사용자는 서비스를 이용하지 못할 수 있다.
따라서 다양한 환경에서의 호환성을 고려해야 한다.

# 6. Glossary

| 용어 | 설명 |
|------|------|
| Smart Medication & Supplement Management System | 본 프로젝트에서 개발하는 시스템으로, 약 및 영양제의 복용 관리와 정보 제공 기능을 수행한다. |
| 사용자 (User) | 시스템을 사용하는 일반 사용자로, 약 정보 조회, 복용 관리, 리뷰 작성 등의 기능을 수행한다. |
| 관리자 (Administrator) | 시스템 운영을 담당하는 사용자로, 사용자 관리 및 리뷰 관리 등의 기능을 수행한다. |
| 약/영양제 | 사용자가 복용하는 의약품 및 건강 보조 식품을 의미한다. |
| 복약 관리 | 약 또는 영양제의 복용 시간, 주기, 기록 등을 관리하는 기능이다. |
| 복용 알림 | 설정된 시간에 약 복용을 사용자에게 알려주는 기능이다. |
| 복용 기록 | 사용자가 약을 복용했는지 여부를 저장한 데이터이다. |
| 리뷰 (Review) | 사용자가 약 또는 영양제에 대해 작성하는 평가 및 경험 공유 내용이다. |
| 데이터베이스 (DB) | 사용자 정보, 약 정보, 복용 기록, 리뷰 등을 저장하는 시스템이다. |
| 서버 (Server) | 데이터 저장 및 처리를 담당하며, 시스템과 사용자 간의 데이터를 관리하는 역할을 한다. |
